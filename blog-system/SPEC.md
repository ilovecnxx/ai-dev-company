# CyberBlog - 未来感博客系统

## 1. 概念与愿景

一个赛博朋克风格的博客系统，仿佛来自2077年的数字世界。深沉的暗色背景上，霓虹光芒勾勒出每一个元素，玻璃态的卡片漂浮在虚空中，每一次交互都伴随着微妙的光晕扩散。内容不再是静态的文字，而是活生生的数字生命体。

## 2. 设计语言

### 美学方向
赛博朋克 × 玻璃态 × 全息界面。参考《银翼杀手2049》的霓虹美学与《攻壳机动队》的数字质感。

### 色彩系统
```
--bg-void: #0a0a0f          // 深空黑背景
--bg-glass: rgba(15, 15, 25, 0.7)  // 玻璃态背景
--neon-cyan: #00f5ff        // 青色霓虹（主色）
--neon-magenta: #ff00ff     // 品红霓虹（强调）
--neon-purple: #a855f7      // 紫色霓虹（次要）
--text-primary: #e0e0e0     // 主文字
--text-dim: #6b7280         // 暗淡文字
--glow-cyan: 0 0 20px rgba(0, 245, 255, 0.5)
--glow-magenta: 0 0 20px rgba(255, 0, 255, 0.5)
```

### 字体
- 标题：`Orbitron` (科技感) + `Rajdhani` (辅助)
- 正文：`Space Grotesk`
- 代码/终端：`JetBrains Mono`

### 动效哲学
- 入场：元素从虚空淡入，带有轻微上浮 (opacity 0→1, translateY 20px→0, 600ms ease-out)
- 悬停：霓虹光晕增强，元素轻微上浮 (scale 1.02, glow intensity +50%)
- 点击：涟漪扩散效果 (ripple from click point, 400ms)
- 滚动：视差深度效果，卡片分层移动

### 视觉资产
- 图标：Phosphor Icons (light weight)
- 装饰：CSS 网格线背景、扫描线效果、噪点纹理
- 边框：1px 霓虹渐变边框 (cyan → magenta)

## 4. 功能与交互

### 核心功能
1. **文章列表** - 玻璃态卡片展示，支持标签筛选
2. **文章详情** - 全息风格的阅读界面，代码高亮
3. **标签系统** - 霓虹标签云，点击筛选
4. **搜索** - 终端风格的实时搜索 (Ctrl+K 唤起)
5. **文章创作** - Markdown 编辑器，带实时预览

### 交互细节
- **卡片悬停**：边框渐变发光，阴影扩散，轻微上浮
- **标签点击**：涟漪效果，页面平滑滚动到筛选结果
- **搜索**：聚焦时背景变暗，模拟终端光标闪烁
- **滚动**：Header 变窄并添加毛玻璃背景

### 边界条件
- 空文章列表：显示「虚空信号」空状态
- 搜索无结果：显示「信号丢失」+ 建议标签
- 文章加载中：骨架屏 + 扫描线动画

## 5. 组件清单

### PostCard
- 默认：暗色玻璃背景，1px 暗边框
- 悬停：霓虹渐变边框亮起，Y轴上浮 4px
- 激活：按下时缩放 0.98

### TagPill
- 默认：半透明背景，文字描边
- 悬停：背景填充，霓虹发光
- 选中：实心填充，强发光

### SearchModal
- 唤起：背景模糊 + 暗化，模态框从顶部滑入
- 关闭：ESC 键或点击外部

### Button
- Primary: 霓虹青色填充，悬停发光扩散
- Ghost: 透明背景，悬停边框亮起
- 禁用：50% 透明度，无交互

### TerminalText
- 文字逐字打印效果
- 光标闪烁 (0.8s 周期)
- 打字声效（可选）

## 6. 技术方案

### 技术栈
- **前端**：原生 HTML/CSS/JS，无框架依赖
- **后端**：Node.js + Express
- **数据库**：SQLite (轻量，随项目)
- **Markdown**：marked.js 渲染

### API 设计
```
GET  /api/posts              - 获取文章列表
GET  /api/posts/:slug        - 获取单篇文章
POST /api/posts             - 创建文章
GET  /api/tags              - 获取所有标签
GET  /api/search?q=keyword  - 搜索文章
```

### 数据模型
```javascript
Post {
  id: integer,
  slug: string (unique),
  title: string,
  excerpt: string,
  content: text (markdown),
  tags: string (comma-separated),
  created_at: datetime,
  updated_at: datetime
}
```
