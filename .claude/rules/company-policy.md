# 公司制度 v1.2

## 一、任务调度

| 任务类型 | 成员 |
|---------|------|
| Python 后端 | python-dev |
| React 前端 | frontend-dev |
| 测试 | qa |
| 安全审计 | security |
| 混合任务 | 按需组合 |

- 同一成员最多并发 2 个任务
- 默认超时: 30 分钟

## 二、Dev-Test 闭环

- QA 主导闭环
- 循环 ≤3 次
- 超过 3 次 → CEO 仲裁

### Bug 优先级
| 优先级 | 定义 | 响应时间 |
|-------|------|---------|
| Critical | 数据丢失/安全漏洞 | 立即 |
| High | 功能严重受损 | 24h |
| Medium | 功能部分受损 | 1周 |
| Low | 体验问题 | 下个迭代 |

## 三、产出强制规范

### API 产出
- [ ] 带 OpenAPI 文档
- [ ] 有错误码定义
- [ ] 单元测试覆盖率 ≥80%

### 前端产出
- [ ] 通过 frontend-checklist.md
- [ ] TypeScript 类型完整
- [ ] 有 Error Boundary

### 安全评审
- [ ] 新功能设计阶段 → 必须触发
- [ ] 依赖更新 → 必须安全扫描
- [ ] 上线前 → 通过 OWASP Top 10

### 测试标准
- [ ] 覆盖率 < 80% → 不得合并
- [ ] Critical/High Bug → 未修复不得上线
- [ ] E2E → 关键路径 100%

## 四、知识共享

- `memory/kb/` - 共享知识库 (只读)
- `memory/members/[name].json` - 个人记忆

### 装备申请
```
[装备申请] 建议新增 MCP/Skill: [名称]
理由: [简短说明]
```

## 五、制度进化

### 触发条件
- 复盘/头脑风暴后
- 同一错误重复 3 次以上

### 进化流程
1. CEO 分析根因
2. 制定改进措施
3. 备份至 `memory/rollback/`
4. 更新本文件
5. 公告全体成员

## 六、禁止事项

- 严禁 CEO 直接写业务代码
- 严禁成员广播 (除紧急安全漏洞)
- 严禁绕过知识库重复造轮子
- 严禁 Critical/High Bug 不上报
- 严禁覆盖率 < 80% 合并
- 严禁绕过安全评审上线
