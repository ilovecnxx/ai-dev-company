# 公司制度 (自动进化版)

> 本文件由 CEO 自动维护，每次复盘或头脑风暴后更新
> 修改前自动备份至 memory/rollback/

## 一、任务调度规则

### 1.1 成员唤醒原则
| 任务类型 | 唤醒成员 | 数量 |
|---------|---------|------|
| Python 后端 | python-dev | 1 |
| React 前端 | frontend-dev | 1 |
| API 开发 | python-dev | 1 |
| 数据库设计 | python-dev | 1 |
| 单元测试 | qa | 1 |
| E2E 测试 | qa | 1 |
| 安全审计 | security | 1 |
| 混合任务 | 按需组合 | - |

### 1.2 并发限制
- 同一成员最多并发 2 个任务
- 超过并发上限时排队等待
- 紧急任务可申请 CEO 特批

### 1.3 任务超时
- 默认任务超时: 30 分钟
- 超时自动挂起，等待 CEO 决策

## 二、Dev-Test 闭环规则

### 2.1 循环次数
- 最多循环 3 次
- 3 次仍未解决 → 上报 CEO 仲裁

### 2.2 Bug 优先级
| 优先级 | 定义 | 响应时间 |
|-------|------|---------|
| Critical | 数据丢失/安全漏洞 | 立即修复 |
| High | 功能严重受损 | 24 小时内 |
| Medium | 功能部分受损 | 1 周内 |
| Low | 体验问题 | 下个迭代 |

## 三、知识共享规则

### 3.1 知识库位置
- `memory/kb/` - 共享知识库 (只读)
- `memory/members/[name].json` - 个人记忆

### 3.2 知识贡献
- 个人踩坑 → 提炼 → 写入 kb/
- KB 更新需 CEO 审批

### 3.3 装备申请
```
[装备申请] 建议新增 MCP/Skill: [名称]
理由: [简短说明]
```
- CEO 审批后安装
- 安装后在 memory/announcements.md 公告

## 四、制度进化规则

### 4.1 进化触发条件
- 复盘会议后
- 头脑风暴会议后
- 同一错误重复出现 3 次以上

### 4.2 进化流程
1. CEO 分析根因
2. 制定改进措施
3. 备份当前版本到 memory/rollback/
4. 更新本文件
5. 公告全体成员

### 4.3 回滚机制
- 可随时回滚到 memory/rollback/ 任意版本
- 回滚命令: `CEO，回滚到 [日期] 版本`

## 五、省 Token 规范

### 5.1 成员行为
- 只读取任务相关文件
- 返回只含状态、产出路径、摘要
- 不夹带全局上下文

### 5.2 CEO 行为
- 记忆检索基于文件名索引
- 不全库扫描
- 只下发微任务上下文

## 六、禁止事项

- 严禁 CEO 直接写业务代码
- 严禁成员之间广播 (除紧急安全漏洞)
- 严禁绕过知识库重复造轮子
- 严禁 Critical/High Bug 不上报
