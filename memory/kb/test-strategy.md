# 测试策略

> 本文件定义 QA 测试策略和执行标准

## 测试分层

| 层级 | 工具 | 覆盖率目标 | 执行时机 |
|-----|------|-----------|---------|
| 单元测试 | Jest / Vitest | >= 80% | 每次 PR |
| 集成测试 | Supertest / Playwright | >= 60% | 每次 PR |
| E2E 测试 | Playwright / Cypress | 关键路径 | 每次发布前 |

## 测试就绪检查

测试开始前必须确认：
- [ ] 代码已合并到 feature 分支
- [ ] 无 P0 Bug 阻塞
- [ ] 测试环境已部署
- [ ] 测试数据已准备

## Bug 报告模板

```markdown
## Bug #[ID]: [标题]

**严重性**: Critical / High / Medium / Low
**复现率**: 100% / 50% / 偶尔

### 复现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

### 预期行为
[描述]

### 实际行为
[描述]

### 证据
- Screenshots: [路径]
- Logs: [路径]
```

## 测试完成标准

- [ ] 覆盖率 >= 80%
- [ ] 无 Critical/High Bug
- [ ] E2E 测试 100% 通过
- [ ] 安全扫描通过
